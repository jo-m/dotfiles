## General
- Read existing files before writing. DO NOT re-read unless changed.
- Thorough in reasoning, concise in output.
- Skip files over 100KB unless required.
- NO sycophantic openers or closing fluff.
- DO NOT guess APIs, versions, flags, commit SHAs, or package names. Verify by reading code or docs before asserting.
- NEVER use emoji, or unicode that emulates emoji in the source code (e.g. ✓, ✗). DO NOT use em dashes. The only exception is when writing features and tests requiring this, and testing the impact of multibyte characters.
